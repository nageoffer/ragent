## 2. 账号与密码（电脑账号/统一身份/应用登录）

### 2.1 典型场景与用户感知

- 新员工入职后出现“电脑可以开机但无法使用公司账号登录”的情况。
- 密码过期或忘记密码导致跨应用登录失败，并出现“多端密码不同步”的困惑。
- 账号被安全策略锁定，常伴随“异地登录提醒”或“频繁输错密码”提示。
- 部门调动/角色变化后权限回收，引发“以前能看到的系统现在提示无权限”的反馈。

### 2.2 自助处理路径（示例）

1. **密码重置**
    - 通过“统一身份自助平台（示例名称）”发起重置流程。
    - 常见验证方式：企业 IM 二次确认 + 绑定手机号或安全密钥（如适用）。
2. **账户解锁**
    - 连续输错密码触发锁定时，优先使用自助解锁；若系统提示存在风险标记，则需服务台协同安全团队复核。
3. **首次登录与身份同步**
    - 若员工信息尚未完成上游系统同步，建议先确认 HR 入职流程是否完成，待身份开通后再次登录。

### 2.3 账号问题排查清单（服务台侧）

- 员工身份类型是否与账号体系一致（正式/实习/外包在权限与有效期策略上通常不同）。
- 是否跨 BU 或跨体系调动导致权限自动回收或账号状态变更。
- 近期是否修改过密码且在多设备端同步失败。
- 是否触发安全风控策略（如高风险地区登录、异常设备指纹、疑似撞库行为）。

### 2.4 工单模板（示例文本）

- 标题：无法登录电脑 / 账号被锁定 / 密码重置失败
- 必填信息：
    - 员工工号与所属部门（用于快速匹配审批链）
    - 设备编号与系统类型（Windows/macOS）
    - 报错截图或文字描述（尽量包含完整提示）
    - 发生时间与网络环境
    - 是否存在紧急交付或业务影响说明

## 3. VPN 支持（远程访问）

> 说明：以下为拟真企业通用写法（阿里巴巴背景风格｜虚构示例），用于 RAG 语料。  
> 实际企业的客户端名称、入口与安全校验方式可能不同。

### 3.1 使用前提（示例）

为保证远程访问安全与稳定性，员工使用 VPN 前通常需满足：

- 已开通公司**统一身份账号**并完成入职身份同步。
- 终端设备符合安全基线（示例要求）：
    - 操作系统版本在公司支持范围内；
    - 已安装终端安全客户端/补丁；
    - 磁盘加密与关键安全策略处于启用状态（如适用）。
- 具备 VPN 使用权限（部分岗位/外包可能需要额外审批）。
- 已通过“公司软件中心/终端自助平台（示例）”安装 VPN 客户端。

---

### 3.2 Windows 连接 VPN（示例步骤）

**适用系统**：Windows 10/11

1. 打开“公司软件中心（示例）”，搜索并安装：
    - **公司 VPN 客户端**
    - 如提示需要依赖组件（安全驱动/网络组件），按指引完成安装。
2. 安装完成后重启电脑（若系统提示）。
3. 打开 VPN 客户端，选择“企业账户登录”：
    - 输入公司账号（工号/邮箱前缀）；
    - 完成二次验证（IM 验证码/短信/安全令牌等，按公司策略）。
4. 在客户端中选择对应的**访问配置/节点**（示例逻辑）：
    - **标准办公访问**（多数员工默认）
    - **研发/高权限资源访问**（需额外授权）
    - **海外/跨区域优化节点**（按合规策略开放）
5. 点击“连接”，等待状态变更为：
    - **已连接 / Connected**
6. 进行连通性验证（示例方法）：
    - 访问“内网连通性检测页（示例）”
    - 或打开常用内部系统进行确认。
7. 若需要访问特定系统仍提示无权限：
    - 说明 VPN 已通，但**系统权限未开通**；
    - 需按对应系统的权限流程另行申请。

**常见小技巧**
- 若你同时使用多个网络工具（如代理/加速器），建议在连接 VPN 前临时关闭，避免路由冲突。
- 出现“证书/认证失败”时，优先检查系统时间是否准确。

---

### 3.3 macOS 连接 VPN（示例步骤）

**适用系统**：macOS 12+（示例）

1. 打开“公司软件中心（示例）”，安装：
    - **公司 VPN 客户端（mac 版）**
2. 首次启动客户端时，系统可能提示授权：
    - **网络过滤/系统扩展**
    - **VPN 配置**
    - 按提示在：
        - *系统设置 → 隐私与安全*
        - *系统设置 → 网络*  
          中完成“允许/启用”。
3. 返回客户端，使用公司账号登录并完成二次验证。
4. 选择访问配置/节点（逻辑同 Windows）。
5. 点击“连接”，确认状态为“已连接”。
6. 验证内网访问：
    - 使用内网检测页或目标内部系统进行确认。
7. 若出现“连接后内网不可达”：
    - 退出 VPN 客户端 → 重新连接；
    - 必要时重启电脑以刷新网络策略。

**mac 常见注意点**
- 升级系统后第一次使用 VPN，可能需要重新授权网络扩展。
- 若使用了第三方网络管理工具或安全软件，可能与 VPN 的网络过滤机制冲突。

---

### 3.4 典型问题与快速排查

#### 3.4.1 无法连接（超时/握手失败）

**优先自查**：

1. 切换网络对照测试：  
   家中宽带 ⇄ 手机热点
2. 校准系统时间与时区（尤其跨时区出差/海外）。
3. 退出客户端重新登录。
4. 关闭第三方代理/加速器。
5. 尝试备用节点或“标准办公访问”配置。

**建议工单字段**：

- 失败截图（含错误码/时间）
- 客户端版本与系统版本
- 所在城市/国家、运营商
- 是否“家宽失败/热点成功”的对照结果

---

#### 3.4.2 连接成功但访问某内网系统失败

**判断要点**：

- 若**多数内网页面可打开**，仅个别系统不可用：  
  通常是**系统权限问题**，应走该系统权限申请或应用故障路径。
- 若**所有内网域名都不可达**：  
  可能是 DNS/路由策略未正确下发。

---

#### 3.4.3 连接后网络变慢

**可能原因**：

- 节点拥塞或跨地域链路抖动
- 终端上同时启用了多种网络代理
- 家用路由器策略/MTU 配置导致丢包

**建议处理**：

- 切换至“就近节点”
- 临时关闭不必要的代理工具
- 使用热点或更换网络环境验证

---

### 3.5 VPN 合规提示（示例）

- 禁止将 VPN 账号转借他人。
- 禁止在未经认可的设备上安装使用。
- 涉及高敏系统访问的权限应设置有效期并接受定期复核。
- 收到异常登录提醒时应立即改密并完成安全确认。

## 4. 邮箱与企业协作

### 4.1 常见请求类型

- 新建、停用或恢复邮箱（与入离职与组织变更强关联）。
- 邮件收发异常（尤其是外部客户邮件丢失或延迟）。
- 容量告警与归档策略咨询。
- 共享邮箱、邮件群组的权限申请与定期复核。
- 钓鱼邮件识别与安全上报。

### 4.2 邮件收发异常排查（员工视角）

**建议排查顺序**：

1. 查看“服务状态公告（示例）”确认是否存在系统级影响。
2. 使用 Web 端与客户端交叉验证：
    - Web 正常、客户端异常 → 优先考虑客户端配置、缓存或网络代理问题。
3. 检查本地规则、黑白名单、转发设置，避免误分流或误删。
4. 若短时间大量群发或对外触达，可能触发安全限流策略。

**需要工单时建议提供**：

- 发件人/收件人（脱敏）
- 邮件主题或关键字（脱敏）
- 发送/接收时间点
- 具体错误提示
- 是否跨域（外部邮箱域名）

### 4.3 共享邮箱与群组权限（示例）

- 共享邮箱申请通常需要：
    - 业务负责人审批
    - 指定共享邮箱管理员
    - 明确使用场景（客服、采购、项目对外联络等）
    - 设置权限有效期与定期复核机制
- 离职/转岗权限回收：
    - 由身份系统触发自动回收
    - 管理员需在周期内检查成员是否与岗位匹配

### 4.4 钓鱼邮件与可疑附件处理

- 不要点击附件或链接；确保不在不可信网络上输入账号密码。
- 通过“安全上报按钮/IM 机器人（示例）”一键转交安全团队分析。
- 若已误点击：
    - 立即断开网络
    - 联系服务台并标记为安全事件
    - 按安全指引完成终端检查、账号改密、会话清理与风险复核

## 5. 园区网络与 Wi-Fi

### 5.1 典型故障画像

- 办公网无法获取 IP 或频繁掉线，影响代码拉取、工单系统与会议。
- 某楼层 Wi-Fi 高峰拥塞，导致视频会议卡顿或音频断续。
- 有线口物理故障或交换机端口策略异常，出现“局部工位不可用”。
- 访客网络可用但办公专网不可用的对比现象。

### 5.2 员工自助排查（办公区）

1. 重新连接 Wi-Fi 或拔插有线网线。
2. 重启网卡或关闭/开启飞行模式以刷新 DHCP 获取流程。
3. 切换 SSID（办公专用/访客网络）进行对照测试。
4. 临时使用手机热点验证是否为终端问题或园区网络问题。
5. 观察“是否同区域多人同时异常”，以便判断影响面与优先级。

### 5.3 工单所需信息（便于网络定位）

- 园区/楼栋/楼层/工位号（越精确越利于现场排查）
- 网络类型（有线/无线、SSID 名称）
- 故障开始时间、结束时间与复现频率
- 影响范围描述（个人、小团队、整层或跨楼层）
- 设备 MAC 地址（如可获取）
- 速度测试与丢包情况截图（如公司允许采集）

### 5.4 访客网络管理（示例）

- 访客 Wi-Fi 仅用于临时外部访问，不保证内部系统可达。
- 重大会议、路演或大型活动可由行政/IT 申请临时扩容或专用 SSID。
- 访客网络带宽与访问策略可能与办公网不同，以降低安全风险。

## 6. 打印机与扫描

> 说明：以下为拟真企业通用写法（阿里巴巴背景风格｜虚构示例），用于 RAG 语料。  
> 实际园区的“打印门户/安全打印队列/刷卡取件”名称可能不同。

### 6.1 打印类型与使用建议（示例）

企业园区常见两类模式：

1. **普通网络打印**
    - 适合日常非敏感资料
2. **安全打印（推荐）**
    - 先把任务发送到安全队列
    - 现场通过工牌/验证码取件
    - 用于包含敏感信息的文件

---

### 6.2 Windows 添加/连接打印机

**适用系统**：Windows 10/11

#### 6.2.1 方式 A：自动发现添加（同网段/园区常用）

1. 确保电脑已连接**办公网络**（有线/办公 Wi-Fi）。
2. 打开：  
   **设置 → 蓝牙和设备 → 打印机和扫描仪**
3. 点击 **添加设备**。
4. 在列表中选择对应打印机名称（通常含楼栋/楼层/区域标识）。
5. 系统自动安装驱动后：
    - 点击“管理” → “打印测试页”。

**适用场景**：  
同楼层或同园区打印机、设备命名规范清晰的区域。

---

#### 6.2.2 方式 B：通过打印门户/队列添加（示例）

1. 打开“公司打印门户（示例）”。
2. 搜索并选择打印机：
    - 按**园区 → 楼栋 → 楼层**筛选
3. 点击 **一键安装/添加队列**。
4. 安装完成后在系统打印机列表中确认可见。

**适用场景**：  
园区打印机多、需要统一驱动模板或安全打印队列绑定的情况。

---

#### 6.2.3 方式 C：手动添加 IP 打印（备用）

1. 打开：  
   **设置 → 打印机和扫描仪 → 添加设备 → 手动添加**
2. 选择：
    - “通过 TCP/IP 地址或主机名添加”
3. 输入打印机 IP（通常由现场贴纸或打印门户提供）。
4. 选择驱动：
    - 优先使用公司推荐驱动
    - 如无，使用通用 PCL/PS 驱动（示例）
5. 打印测试页。

**适用场景**：  
自动发现失败、或需要连接特定专用设备。

---

### 6.3 macOS 添加/连接打印机

**适用系统**：macOS 12+

#### 6.3.1 方式 A：系统自动发现（Bonjour/AirPrint）

1. 确保连接办公网络。
2. 打开：  
   **系统设置 → 打印机与扫描仪**
3. 点击 **添加打印机**。
4. 在“默认”页签中选择目标设备。
5. 驱动选择：
    - 优先 **AirPrint** 或公司推荐驱动
6. 添加后打印测试页。

---

#### 6.3.2 方式 B：通过 IPP/LPD 手动添加（备用）

1. **系统设置 → 打印机与扫描仪 → 添加打印机**
2. 切换到 **IP** 页签：
    - 地址：输入打印机 IP
    - 协议：选择 **IPP**（优先）或 **LPD**
3. 驱动选择：
    - 公司推荐驱动
    - 或“通用 PostScript/PCL（示例）”
4. 完成添加并测试。

---

#### 6.3.3 方式 C：安全打印队列（示例）

若园区启用安全打印：

1. 在打印门户中选择“安全打印队列（示例名称）”并按指引安装。
2. 打印时选择该队列。
3. 到打印机现场刷工牌/输入验证码释放任务。

---

### 6.4 扫描功能（示例）

常见扫描输出方式：

- **扫描到邮箱**
- **扫描到共享文件夹/云盘入口（按权限）**

**使用建议**：

1. 在打印机面板选择“扫描”。
2. 选择目标：
    - 扫描到本人邮箱
    - 或选择已授权的共享目录
3. 若扫描到邮箱失败：
    - 检查邮箱是否容量告警
    - 确认是否输错邮箱地址或选择了错误域

---

### 6.5 常见问题与排查

#### 6.5.1 搜不到打印机

- 确认已连接**办公网络**而非访客网络。
- 关闭并重新打开打印机列表刷新。
- 尝试：
    - Windows：用“手动添加/IP 添加”
    - macOS：用“IP 页签 + IPP”方式
- 若同区域多人无法发现：
    - 可能是打印服务或网络策略问题，建议提单。

---

#### 6.5.2 打印排队不出纸

- 清理本地队列中异常任务。
- 小批量测试（1–2 页）验证。
- 换同机型不同逻辑队列（如存在）。
- 若多人同时失败：
    - 说明更可能是设备或打印服务故障。

---

#### 6.5.3 卡纸/缺墨/耗材

- 依据设备提示进行简单处理（如打开侧盖取纸）。
- 无法处理时，联系楼层行政或现场 IT 支持。

---

### 6.6 打印合规提示（示例）

- 含敏感信息的文件建议使用**安全打印**。
- 打印后及时取件，避免资料在公共区域遗留。
- 禁止使用私人 U 盘等介质直接拷贝打印敏感数据（如公司有相关限制策略）。

## 7. 办公电脑与终端设备

### 7.1 常见硬件/系统问题

- 无法开机、蓝屏、系统更新后循环重启。
- 磁盘空间不足导致开发环境或办公软件异常。
- 风扇噪音明显、机身过热影响稳定性与电池寿命。
- 摄像头/麦克风不可用影响远程会议与客户沟通。

### 7.2 服务策略与员工建议（示例）

- 若影响当天重要会议或发布节奏，可申请**临时备用机**确保交付。
- 硬件疑似损坏时优先现场检测，必要时走维修或换机流程。
- 性能类问题通常先执行系统健康检查、清理启动项与缓存，再评估是否需要配置升级。

### 7.3 终端安全基线（示例语境）

- 对关键岗位设备启用系统加密与强制补丁策略。
- 禁止安装未经认证的软件或高风险驱动。
- 外接存储设备可能触发审批或安全扫描机制。

---

## 8. 办公软件（IM、文档、会议、浏览器组件）

### 8.1 常见请求类型

- 通过“公司软件中心（示例）”安装或升级办公套件。
- 软件激活失败或授权状态异常。
- 插件冲突导致文档打不开、闪退或格式错乱。
- 会议客户端无法入会、音视频延迟高、共享屏幕失败。

### 8.2 自助建议（员工端）

1. 优先使用公司软件中心进行安装，避免第三方来源造成版本不一致与安全风险。
2. 清理缓存并重新登录，以解决偶发的鉴权与配置下发异常。
3. 团队协作时建议统一主版本，减少因文档格式/插件差异导致的协作问题。

### 8.3 会议问题快速排查

- **音频异常**：
    - 检查系统输入/输出设备选择是否正确。
    - 关闭其他占用音频通道的软件（浏览器标签页、录屏工具等）。
- **视频异常**：
    - 排查摄像头是否被权限管理或其他应用占用。
- **屏幕共享失败**：
    - 检查操作系统权限（如 macOS 的“屏幕录制”权限）。
    - 临时关闭可能冲突的安全软件模块并按合规要求重新启用。

## 9. 权限申请与系统访问（与 IT 交叉）

### 9.1 典型场景

- 新项目启动时需要批量开通研发与协作平台权限。
- 外包人员需要临时访问受控系统完成特定交付。
- 团队成员因角色升级或职责扩展申请更高权限。

### 9.2 申请规则（示例）

- 业务系统权限由业务 owner 负责审批与定期复核。
- 基础 IT 与安全类权限由 IT/安全团队结合风险进行核验。
- 临时权限必须设置有效期，到期自动回收以降低长期风险。
- 批量申请需提供人员清单、岗位角色、业务说明与成本中心信息（如适用）。

## 10. 故障等级与升级机制（示例）

### 10.1 等级参考

- **P1**：
    - 大面积办公网络不可用、核心远程访问通道不可用、邮件基础服务中断。
    - 通常需要启动应急沟通群、公告机制和跨团队联动。
- **P2**：
    - 单园区或单楼层网络异常、关键业务团队无法使用协作工具或 CI/CD 依赖环境。
- **P3**：
    - 单人终端、单点软件、常规账号请求或打印问题。

### 10.2 升级路径（示例）

1. 服务台接单并完成初步分诊与信息补全。
2. 15 分钟内无法定位或影响范围扩大，升级二线专业团队。
3. 若出现多点同时反馈或疑似系统级问题，启动应急响应与公告。

## 11. FAQ（综合增强版）

### Q1：我在家连不上 VPN，但手机热点可以连，是我电脑问题吗？

A：不一定。更常见的情况是**家用宽带路由策略、运营商链路或本地网络设备**导致加密隧道不稳定。  
建议先用热点保障紧急工作，再提交工单说明**运营商、所在地、客户端版本、失败截图**及“热点可用/家宽不可用”的对照结果，便于网络团队快速定位。

### Q2：邮箱突然收不到外部客户邮件怎么办？

A：先检查垃圾箱、规则与自动转发设置，再用 Web 端与客户端交叉验证。  
若确认缺失或明显延迟，提交工单并附**时间范围、主题关键字、客户域名（脱敏）**等信息，协作平台团队可按合规范围进行投递日志核查。

### Q3：打印机显示正常但不出纸？

A：建议先清理本地打印队列，尝试小批量打印并切换逻辑队列。  
如果同区域多人同时异常，通常更可能是设备端或打印服务问题，应按“打印设备故障”提单并写明设备位置与编号。

### Q4：我忘记电脑密码，能让同事帮我重置吗？

A：不可以。密码重置必须走**本人自助或服务台身份核验**流程，  
这样能避免账号被冒用并满足企业信息安全审计要求。

### Q5：会议共享屏幕提示无权限？

A：常见原因是操作系统权限未开启，尤其是 macOS 的“屏幕录制权限”。  
按系统提示授权后重启会议客户端通常可恢复；若仍失败，建议提供系统版本与截图提单由终端团队协助。

## 13. 员工自助诊断清单（增强版）

方便你做“检索 → 逐步指引”的结构化语料：

- **VPN 连接失败**：  
  优先切换网络对照测试、校准系统时间、关闭第三方代理、尝试备用节点，并保留完整失败截图与版本信息。
- **邮箱异常**：  
  先用 Web/客户端交叉验证，检查规则与垃圾箱，再按合规要求提供时间范围与脱敏主题关键字。
- **Wi-Fi/有线网络问题**：  
  记录准确位置与影响范围，进行 SSID 对照测试，必要时提供速度测试与丢包信息。
- **打印排队**：  
  先清队列、小批量验证、切换逻辑队列；多人同时异常时优先报设备或打印服务故障。
- **密码与账号**：  
  必须本人自助或身份核验重置，禁止代操作与账号共享。
- **会议共享失败**：  
  优先检查系统权限、设备占用与安全软件冲突项。

## 14. 术语表（增强版）

- **IT 服务台**：  
  负责统一入口受理、问题分级、信息补全、常见故障处理与跨团队升级协调的一线团队。
- **二线团队**：  
  具备专业技术能力的支持团队，如网络、终端、协作平台、安全等，通常负责复杂问题定位与根因治理。
- **统一身份**：  
  用于管理员工账号生命周期、登录与权限的统一体系（示例概念）。

  公司内网或受控分发渠道，用于保障软件版本一致性与合规安装（示例概念）。
- **安全打印**：  
  需要身份验证后才可取件的打印模式，用于降低敏感信息在公共区域泄露的风险。

## 15. 可用于 RAG 的“问答对”样例（增强版）

1. **问**：我在家里用宽带连不上 VPN，但热点可以，应该怎么写工单？  
   **答**：请写明“家宽失败/热点成功”的对照结果，并附运营商、所在地、客户端版本、系统版本、失败截图和发生时间窗口。  
   这类信息有助于网络团队判断是家宽链路、路由策略还是节点侧策略导致问题。

2. **问**：共享邮箱的成员权限需要多久复核一次？  
   **答**：示例做法通常要求管理员定期复核（如每 6 或 12 个月），并在成员离职/转岗时由身份系统触发自动回收，  
   以确保权限始终与岗位职责匹配并降低长期安全风险。

3. **问**：打印任务一直排队不出纸，如何快速定位是我电脑还是打印机问题？  
   **答**：先清理本地队列并用小批量测试；如果同区域多人同时异常，优先怀疑打印设备或打印服务。  
   工单中最好附设备位置、编号与是否多人受影响的描述。

4. **问**：会议共享屏幕失败但音视频正常，最可能的原因是什么？  
   **答**：常见原因是操作系统权限未开启或被安全软件模块拦截，尤其是 macOS 的“屏幕录制权限”。  
   先按系统提示授权并重启客户端，仍不行再提单并附系统版本与截图。
